📐 Methodology — why daily bars, not tick data (fully disclosed)
A genuine 5-year intraday (minute-level) backtest is not feasible here — free-tier data providers, including the one this dashboard uses, only retain intraday bars for a recent rolling window (days to a few months), never 5 years. Rather than quietly fabricate a precision this scan doesn't have, here is exactly what is simulated instead: an Intraday trade enters at the next session's open and MUST exit that same session — target/stoploss are counted as hit if that day's High/Low actually reached the level, otherwise the position is force-squared-off at that day's Close (exactly like a real intraday MIS position). A BTST trade enters at today's Close and may be exited any time up to tomorrow's Close — an opening gap straight through the level, an intraday High/Low touch, or a forced exit at tomorrow's Close if neither triggers. Both use the real High/Low of the relevant session, so the simulation is not blind to intraday extremes — it simply cannot know the exact intraday timing/path, which is exactly why this is disclosed here rather than presented as tick-precise. Fixed risk-reward: 1:2.5 for Intraday and BTST alike. Same-rule guarantee: the identical entry rule generates both today's live call and every historical backtest trade below — SELL (short) setups use a fully mirrored bearish scoring system (bearish engulfing, shooting star, bearish marubozu, LL-LH structure, gap-down-on-volume) rather than an inverted BUY score. Confidence % blends the Wilson-shrunk win-rate (sample size aware), the day's own Setup Score, resistance/support realism against the fixed target, live volume confirmation, price-action strength, and the Nifty 50 bias described above.
